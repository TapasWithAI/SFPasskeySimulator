State 1 — What the Customer Sees
The customer has been prompted by Salesforce to register a passkey (likely first-login MFA enforcement). They are on the native Salesforce passkey registration page at their custom domain.
Engineer Talking Points
✓ Passkeys are phishing-resistant — Unlike SMS OTP or TOTP codes, passkeys use public-key cryptography. The private key never leaves the device. Reassure customers this is more secure than their old method.
Two paths from this screen:
1. Register Passkey — Triggers the browser's native WebAuthn dialog (State 3)
2. Choose Another Method — Opens the MFA selector (State 2)
Quick Script — Passkey Registration 
Hi [Customer Name],

I can see you're on the Salesforce passkey registration page. Let's get this set up for you.

You'll see a page with "Register a Passkey" at the top. Click the blue "Register Passkey" button, and your browser/device will pop up a dialog asking you to verify your identity — this might be Face ID, Touch ID, or a PIN depending on your device.

If you'd prefer a different method (like an authenticator app), click "Choose Another Verification Method" at the bottom instead.

Let me know which option you'd like to proceed with!
State 2 — MFA Method Selection
The customer chose to not use a passkey. They are now selecting an alternative MFA method. Guide them to the appropriate option based on their device and preference.
Option Guidance Matrix
Option 1 — Built-in authenticator: Best for customers who want the passkey experience but clicked the wrong button. Redirects back to the WebAuthn passkey flow.
Option 2 — Salesforce Authenticator App: Best for mobile-first users who are already using the SF ecosystem. Requires app install on iOS/Android.
Option 3 — TOTP (Google Auth / Authy): Use for customers on locked-down devices who cannot install SF Authenticator, or who already have a TOTP manager set up.
Script — Method Selector 
Hi [Customer Name],

You should now see a "Choose a Verification Method" screen with three options.

Quick guide:
• If you want the easiest option: select "Built-in authenticator" — this uses Face ID, Touch ID, or Windows Hello on your device.
• If you have a smartphone and want push notifications: select "Salesforce Authenticator app" and download it from the App Store or Google Play.
• If you already use Google Authenticator or Authy: select the third option and scan the QR code when prompted.

Which option sounds best for your setup?
State 3 — WebAuthn Browser Dialog
This is the native browser dialog — it is NOT a Salesforce UI. It is controlled by the OS/browser. Customers may be confused because it looks different from Salesforce branding. This is expected and secure behavior.
Common Issues at This Step
Dialog doesn't appear: Browser may be blocking popups or WebAuthn is disabled. Check browser flags — Chrome: chrome://flags, ensure "WebAuthentication" is not disabled.
Biometric prompt fails: Customer may need to set up Windows Hello / Face ID at the OS level first before the browser can use it.
"Save another way" button lets customers choose where to store the passkey. Clicking it goes to State 4.
Script — WebAuthn Dialog 
Hi [Customer Name],

After clicking "Register Passkey," you should see a pop-up dialog from your browser (not Salesforce) titled "Create a passkey for tapas-demo.my.salesforce.com."

This is completely expected — it's your browser securely creating the passkey. You'll see your email address shown to confirm it's the right account.

Script — iCloud Keychain Path 
Hi [Customer Name],

You clicked "Save another way" and should see two options.

For the best experience on your Apple device, select "iCloud Keychain." This stores your passkey securely in iCloud and syncs it to all your Apple devices (iPhone, iPad, Mac) automatically.

Requirements:
• You must be signed into iCloud on this device
• iCloud Keychain must be enabled in Settings > [Your Name] > iCloud > Passwords & Keychain

After selecting it, you'll be prompted to verify with Face ID, Touch ID, or your device passcode.

Once saved, you can log into Salesforce from any Apple device using the same passkey!
Script — Chrome Profile Path 
Hi [Customer Name],

If you're using Chrome on Windows or prefer Chrome across devices, select "Your Chrome profile."

This stores the passkey in your Google Chrome profile synced to your Google Account. You'll be able to use it on any device where you're signed into the same Chrome profile.

Requirements:
• You must be signed into Chrome with your Google Account
• Chrome sync must be enabled for passwords/passkeys

Note for Windows users: Chrome will use Windows Hello (PIN, fingerprint, or face) to verify you when creating or using the passkey.

After selecting Chrome profile, follow the on-screen prompt to verify your identity. The passkey will be saved and ready to use immediately!
